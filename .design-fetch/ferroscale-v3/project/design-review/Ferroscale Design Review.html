Design review · Ferroscale v3

Finishing the V3 — making it feel like an app, not a layout.

You're 90% there. The bones (warm cream palette, terracotta accent, Space Grotesk + IBM Plex Mono, premium-* control system) are strong. What's left is the last 10% — the tiny rhythm and behavior choices that separate a beautifully styled web page from a real product. This review walks the app section-by-section with concrete fixes, ranked.

TL;DR — the 8 things that buy you the most native feel

Each of these is small in scope but has compound visual impact. Ship these first and the app will feel meaningfully different.

Priority order

  1. Consolidate the panel/button vocabulary down to ~4 primitives. You currently have panel-base, panel-unibody, panel-raised, panel-emphasis, premium-control, premium-segment, premium-action-button, premium-icon-button + ad-hoc card-*. Eight surfaces is style sprawl — pick four (Surface, Raised, Inset, Emphasis) and one button family with size variants. Visual rhythm tightens immediately.
    M · 3-4h
  2. Lock the uppercase-label tracking to one value (0.10em). I count at least 0.06, 0.08, 0.10, 0.12, 0.14 in use for the same micro-caps role across mobile/desktop. Pick one. Same for label font-size — --font-size-2xs (0.625rem) is your token; use it everywhere.
    L · 1-2h
  3. Make the create-thing CTA the same across Saved, Projects, Compare. Saved's "New preset" is a black foreground button. Projects' "+" is an accent circle. Same role — adopt one. Recommendation: solid --surface-inverted for the primary "create" everywhere, the accent for state-bearing actions (live, in-progress, primary success).
    L · 1h
  4. Search field that doesn't search is worse than no search. Desktop Saved has a visual-only ⌘K search affordance. Users will type. Either wire it to a live filter on entry.name + profileLabel + grade + tags or hide it until shipped. Same applies to the ⌘K hint in projects.
    M · 2-3h
  5. Allow text selection on result numbers. Global user-select:none on html means the result weight/price can't be copied. That's the #1 reason a sales engineer opens this app — let them copy 347.82 kg. Mark .tabular-nums result values + .select-text and they're back.
    L · 30m
  6. Header rhythm: mobile gets a clean app bar, desktop gets a 48px workstation topbar — make them feel like one family. Today mobile has a tall blurred sticky header with a tiny 4px profile icon inside an inset square (looks broken). Desktop tabs each invent their own topbar. Pick one topbar grammar: logo · title · slim context chip · search · primary CTA · settings. Bring mobile's down to 40px and make the context chip its own element below if needed.
    M · 3-4h
  7. Numpad weight display: cap to fit, and reflect units. The 60px result number on mobile breaks layout when weight ≥ 10,000 kg (5 chars + decimal) on iPhone SE-width devices. Either auto-shrink to fit container (clamp(36px,12vw,60px)) or switch to t at >1000 kg with a tiny "kg → t" hint. Bonus: the current "kg" suffix doesn't change when user is in imperial.
    M · 2h
  8. Add a real focus / hover / pressed system for the field strip + chips. Today: active chip = accent surface (good). Hover on desktop = nothing visible. Focus-ring = global accent outline at 3px, looks heavy-handed on small chips. Add a --ring-soft: 0 0 0 3px color-mix(in srgb, var(--accent) 22%, transparent) token, use it on all interactive surfaces, drop the global hard outline.
    M · 2h
02

Design system — consolidation

A handful of tokens are doing 90% of the work already. The other 10% is fighting itself. Three concrete consolidations:

Fix

Surface ladder is too crowded

You ship 8 surface/panel primitives. Three of them — panel-base, panel-unibody, panel-raised — render at nearly the same elevation in light mode; users can't tell them apart and you can't stay consistent across screens. Collapse to a four-step ladder:

  • Surface (default, what cards sit on) — flat, 1px border
  • Raised (interactive card, hover target) — soft shadow, no border
  • Inset (grouped section background, secondary fills) — slightly darker, no shadow
  • Emphasis (the chosen / selected / focused thing) — accent-tinted surface, accent border

Same for buttons. Today you've got premium-action-button, premium-icon-button, premium-segment, plus inline-styled CTAs everywhere. One button family with size = sm|md|lg and tone = neutral|primary|ghost|danger kills 80% of the special cases.

Surface--surface
Raised--surface-raised
Inset--surface-inset
Emphasis--surface-emphasis
Accent surface--accent-surface
Inverted--surface-inverted
Action

Add a docs/DESIGN_TOKENS.md with the four-step ladder + button family rules. Refactor opportunistically — don't bulk-replace; replace as you touch components.

Fix

Uppercase-label tracking is all over the place

Same micro-caps role uses 0.06em, 0.08em, 0.10em, 0.12em, 0.14em in different files. The eye reads inconsistent line-up across cards. Pick one — I'd use 0.10em at 11px for all "section eyebrow" labels and 0.06em at 10px for chip-internal labels.

Fix

Focus-ring is too loud

Global :focus-visible { outline: 3px solid var(--focus-ring); outline-offset: 2px } looks aggressive on small chips, numpad-style keys, and inline edits. Replace with a soft ring token:

--ring-soft: 0 0 0 3px color-mix(in srgb, var(--accent) 22%, transparent);
:focus-visible { outline: none; box-shadow: var(--ring-soft); }

Keeps the affordance, loses the harshness. The Done key on the numpad is a good test case.

Investigate

The actual type roles in your app cluster into 7 sizes. They aren't documented anywhere. Lock them down and components stop reinventing scales:

Fix

Result number cannot be copied

Fix

Result number overflows past 4 digits

At 3.75rem (60px) the result eats horizontal space; 1,234.56 at iPhone SE width clips the kg suffix in dev. Two options:

  • Switch units automatically. ≥ 1000 kg → display in t with 2 decimals + a "t" suffix, keep the kg in a small line below.
  • Or fluid-shrink. font-size: clamp(36px, 12vw, 60px) on the digit span; the layout doesn't move when the value grows.
Fix

"Live" badge persists when nothing's computed

The green-dot "Live" badge appears on the result card even before the user has entered anything. Map it to result state: result ? Live : Idle, or hide entirely until first result lands.

Win

Numpad active-scale is the right amount

The 0.96 active scale on .key combined with duration-75 feels good — keep this as the canonical "tactile pressable" combo and propagate it to chips and other touch surfaces, replacing the global 0.985.

Investigate

Add an undo / redo affordance you can actually see

You ship UNDO/REDO in the calc reducer + keyboard shortcuts, but mobile users have no visible entry point. A subtle ⤺ button on the result card when canUndo, animated in, would surface the feature for the 95% of users who'll never know the shortcut exists.

Fix

Profile + material chip cards: corner family

Profile/Material chip cards use rounded-2xl (16px). FieldChip uses rounded-[0.85rem] (~13.6px). Profile glyph inside the chip uses rounded-[0.65rem]. Result card uses rounded-[1.4rem]. Numpad uses rounded-[1.25rem]. Five different radii in 200 vertical pixels.

Lock down a corner ladder and apply consistently:

--radius-pad: 8px;     /* inner icon glyphs */
--radius-control: 14px;/* fields, chips, single-line controls */
--radius-card: 20px;   /* result, numpad, chip cards */
--radius-screen: 28px; /* outer modal, full-bleed sheet */

Mobile navigation

You removed the bottom tab bar — gutsy and the right call for a focus app. But the hamburger menu is now doing all the nav work, and a single tap target for six destinations is a lot to ask.

Investigate

Add a swipe-back gesture to the menu sheet

The MobileMenuSheet opens from a corner button. iOS users will swipe to dismiss — make sure vaul is configured with horizontal drag dismiss as well as the existing vertical one.

Fix

Header icon density

Theme toggle + menu live in the mobile header today. On Projects detail you add a third "more" button. The header gets crowded with low-frequency buttons. Move theme into the menu sheet; theme is a settings-level decision, not a header-level one.

Investigate

Quick-jump pill

Consider a small floating action — bottom-right, 56px, accent — that opens Quick Calc. The most powerful feature in your app currently lives behind ⌘K on desktop and nothing on mobile. A FAB earns its keep here.

Win

App-locked html class is a great touch

The app-locked trick to kill iOS rubber-band on the calculator route is exactly the kind of detail that separates a web app from a native one. Keep it. Document it in AGENTS.md so future-you doesn't accidentally undo it.

07

Saved presets

Solid table. Real friction is in the affordances around it.

Critical

Search bar that doesn't search

The ⌘K search field on the Saved topbar is a non-functional placeholder. Either:

  • Wire it. Filter saved by name + profileLabel + gradeLabel + tags. Debounce 80ms. Highlight matched substrings.
  • Or remove. The filter chips below already do the job; the empty search is more confusing than useful.
Fix

Row menu uses native confirm

window.confirm(t("mobileSaved.confirmRemove", { name: entry.name })) ships a system dialog that breaks the app's visual world. Build a small ConfirmDialog primitive: same panel-base, red destructive button, fade-in 120ms. Reuse for project delete + clear history.

Fix

"Updated" column reads too quietly

The formatRelative output is muted text; users scan for it. Bold "just now" / "1m ago" within the first hour so recent work surfaces. Beyond 1 hour, fall back to muted absolute date.

Investigate

Multi-select for bulk actions

The table is begging for it. Hold shift, click rows, bulk add to project / bulk delete / bulk export CSV. A shift-click multi-select plus a bottom action bar that slides up — very pro feel.

Fix

Row hover is invisible in dark mode

hover:bg-surface-raised in dark theme is ~3% lighter than --surface. Almost imperceptible. Either bump the hover state to --surface-emphasis in dark or add a 1px border-color: var(--border-strong) on hover.

Win

Inline rename on double-click is exactly right

Power-user move. Keep it. Add a small "rename" pencil that fades in on row hover for users who don't know the gesture.

Projects

Two-pane is the right call. A few rough edges:

Fix

"Create project" CTA mismatch

Projects' "+" is an accent-orange circle; Saved's "New preset" is a black pill. Same role. Pick one — I'd use --surface-inverted as the canonical primary CTA across the app, and reserve accent for state and emphasis.

Fix

Active project chip is louder than the rest

Active project gets border-accent-border + bg-accent-surface; inactive items get no chrome at all. Add a soft hover row-fill on inactive items and a left accent bar on active to match sidebar idiom. Same active treatment, one place.

Investigate

Project totals deserve more weight

Aggregates (computeAggregates) total weight / cost / parts is high-signal but currently lives as small muted text. Promote to a hero strip: 2,847 kg · 14 parts · € 4,210, with a small trendline if you have history.

Fix

Empty project state too plain

The rail empty state is two lines of muted text. Add a stamp-like illustration (the existing accent-square pattern from Saved empty) and a "Create your first project" CTA — empty rail is your onboarding moment.

Investigate

Drag rows between projects

HTML5 drag, vaul-friendly. Moves a calculation from one project to another. Even if you start with desktop-only, it's the kind of detail that makes the app feel finished.

Pin to compare from anywhere

Compare is the kind of feature users won't discover on their own. The little "pin" button in the result panel is your hook — make sure it's visible from the saved row hover state too, not just the active calc.

10

Settings

Settings will be the unloved screen. Two ways to keep it tight:

Fix

Group + section divider grammar

.form-group + .form-group uses a 1px top border. Section titles are H-style inline. Settings would feel more iOS-app if it switched to grouped lists: each block has its own card with a small caption above it ("DISPLAY", "DATA", "PRIVACY"). Native settings rhythm beats web-form rhythm here.

Fix

Reset-all needs friction

It's nuclear. Currently resetAll() dispatches RESET_ALL immediately. Wrap in a typed confirm: user types RESET to enable the button. Annoying — but exactly as annoying as the action deserves.

Investigate

Sync status as a real first-class block

You have a full Google-backed sync. The settings sync section should look like an Apple iCloud screen — big tile up top showing connected state, last sync time, snapshot count; expand for advanced controls. Today it reads as a list of toggles.

Win

Theme + text size live together

Bundling theme, text size, default unit at the top of settings is the right call. Don't move them.

11

Empty states + first-run

Today empties are inconsistent: Saved has a centred icon-card + CTA; Projects rail has a 2-line text-only state; Result has an EmptyState primitive. Pick one.

Fix

One empty-state primitive

Promote Saved's empty to the canonical pattern and use it everywhere:

  • Icon block — 96px rounded square in accent-surface with the section icon at 40px in accent-text.
  • Headline — 24px, tracking-[-0.02em], bold.
  • Body — 14px, foreground-secondary, max 60ch, balanced wrap.
  • CTA — --surface-inverted primary, 44px tall, arrow tail.
  • Quiet helper link — "Or try a sample" / "Learn more" — small, muted, underlined.

Every empty state in the app uses these five rules and only these. No exceptions for "small" empties — they get the small-variant of the same composition.

Investigate

Onboarding nudge: prefill a sample

The existing OnboardingFlow is a stepper. Most installed PWA users will skip it and hit a blank calculator. Drop a one-time accent banner on the calc on first visit: "Try a sample: HEA 120 × 6 m, S235JR." One tap loads it. Dismisses on first calc.

Investigate

Install prompt has a home

Per your roadmap (FEATURE_IMPROVEMENTS.md P1.1), you want a PWA install CTA. Don't put it in the topbar — put it as the bottom-most item in the sidebar footer ("Install Ferroscale") and as a one-time accent banner on the settings tab the first time it's reachable. Browser-aware, dismissible.

Fix

Skeleton loaders are designed but unused

You ship a .skeleton shimmer style — but the only place I see it consumed is loosely. Wire it into the project rail load + saved list load, even though the data is local — a 100ms shimmer on initial paint sells the "app, not a webpage" feeling for free.
